<button>: The Button element

The <button> HTML element is an interactive element activated by a user with a mouse, keyboard, finger, voice command, or other assistive technology. Once activated, it then performs a programmable action, such as submitting a form or opening a dialog.

By default, HTML buttons are presented in a style resembling the platform the user agent runs on, but you can change buttons' appearance with CSS.

Attributes

This element's attributes include the global attributes.

autofocus

This boolean attribute specifies that the button should have input focus when the page loads. Only one element in a document can have this attribute.

autocomplete

This attribute on a <button> is non-standard and Firefox-specific. Unlike other browsers, Firefox persists the dynamic disabled state of a <button> across page loads. Setting autocomplete="off" on the button disables this feature.

disabled

This boolean attribute prevents the user from interacting with the button: it can not be pressed or focused.

Firefox, unlike other browsers, persists the dynamic disabled state of a <button> across page loads. Use the autocomplete attribute to control this feature.

form

The <form> element to associate the button with (form owner). The value of this attribute must be the id of a <form> element in the same document. (If this attribute is not set, the <button> is associated with its ancestor <form> element, if any.)

This attribute lets you associate <button> elements to <form> elements anywhere in the document, not just inside a <form>. It can also override an ancestor <form> element.

formaction

If the button is a submit button (a <form> element is ancestor or associated with it, and it does not have type="button"), this attribute specifies how to encode the form data that is submitted. If specified, this attribute overrides the enctype attribute of the buttons's form owner. Possible values:

  • application/x-www-form-urlencoded: The default if the attribute is not used.
  • multipart/form-data: Use to submit <input> elements with their type attributes set to file.
  • text/plain: Specified as a debugging aid; it should not be used for real form submission.
formmethod

If this button is a submit button (a <form> element is ancestor or associated with it, and it does not have type="button"), this attribute specifies the HTTP method used to submit the form. If specified, this attribute overrides the method attribute of the button's form owner. Possible values:

  • post: The form data are included in the body of the HTTP request when sent to the server. Use when the form contains information that should not be public, like login credentials.
  • get: The form data are appended to the form's action URL, with a ? as separator, and the resulting URL is sent to the server. Use this method when the form has no side effects, like search forms.
